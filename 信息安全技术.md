## 一、信息安全概述
### 1.1、信息安全的定义
+ 为数据处理系统建立和采用的技术与管理的安全保护
+ 保护计算机硬件、软件和数据不会因偶然个人恶意的原因遭到破坏、更改与泄露
###  1.2、信息安全的基本属性
+ 概述：保密性、完整性和可用性
	1. 保密性：确保在存储、使用和传输的过程中不会泄露给非授权的用户与实体；<font>信息的保密性主要通过加密技术确保</font>
	2. 完整性：确保用户信息在存储、使用、传输过程中不被非授权用户篡改
	3. 可用性：确保授权用户或者实体对于信息及资源的正常使用不会被异常拒绝，允许其可靠而且及时的访问信息以及资源。
### 1.3、信息安全威胁表现形式
+ 信息泄露：信息被泄露给某个非授权的个体
+ 破坏信息的完整性：即对信息非授权的增删改查
+ 拒绝服务：对信息或其他资源的合法访问被无条件的阻止

## 三、对称密码体系
### 3.1、定义
+ 特征数据加密与解密使用同一个密钥
+ 安全性依赖于：
	1. 强度应该满足敌手已知发现密钥
	2. 发送方和接收方必须以安全的方式传递和保护密钥副本
### 3.2、流密码
+ 流密码加密时先将文本、声音、图像等原始明文转换为01字符串
+ 将它与密钥异或生成流密文流发送给接收者，然后接收者与相同的密钥逐位异或得到原始密文
+ 将明文分成一定长度的分数组，分别对各个分组用同一密钥流序列的不同部分加密产生响应的密文
### 3.3、对称算法的优点
+ 加密解密处理快，保密度高
### 3.4、对称算法的缺点
+ 密钥是保密的关键，发信者必须安全、妥善的把密钥护送到收信方不可泄露内容
+ 密钥需要安全信道，成本高
+ 多人通信：密钥组合的数量会出现爆炸式增长，算法分发过于复杂，所花代价高，通信过于复杂
+ 双方必须统一密钥，才可发送保密信息
+ 存在数字密钥签名困难问题
### 3.5、DES算法
#### (1)、特点
+ DES算法以64位为分组单位镜像分组。DES中密钥长度为56位
#### (2)、设计思想
1. 64位明文进行初始置换IP，用于重排明文分组的64比特数据
2. 顺序经过16轮功能相同的变换，每一轮的变换输入是上轮变换输出和右部56位初始密钥生成的48位子密钥k
3. 将第16轮变换输出的64位进行左右32位变换，然后对其进行逆初始置换
### 3.6、IDEA
#### (1)、解释
+ IDEA：国际数据加密算法；64/128
+ AES：高级加密标准；128/16，192/24，256/32
+ DES：数据加密标准；64/56
### 3.7、分组密码的工作模式
#### (1)、原理
1. 将明文按照固定长度分组，通过加密算法进行块级加密，需要一个密钥作为参数
2. 先按照固定长度进行分组，通过相同的密钥和算法进行解密
#### (2)、工作模式
1. 电码本模式（ECB）：将明文数据分组后，每个分组都使用相同的密钥进行加密，因此不同分组之间不存在依赖关系，加密结果也是独立的。
2. 密码分组链接模式（CBC）：将明文数据分组后，每个分组使用密钥加密后，再将上一分组的加密结果与当前分组进行异或操作，产生下一个分组的加密输入，从而引入了分组间的依赖关系，提高了安全性。
3. 密码反馈模式（CFB）：将密钥通过加密操作生成一个系列的密钥流，然后将明文数据分组后，按位进行加密。
4. 输出反馈模式（OFB）：通过将密钥通过加密操作生成一个系列的密钥流，但它是将前面加密结果输出，并循环移位后作为下一分组的输入，因此不存在依赖关系。
5. 计数器模式（CTR）：将密钥通过加密操作生成一个系列的密钥流，但使用不同的计数器值作为加密输入，而非前一分组加密结果，因此不存在依赖关系。
### 3.8、公钥加密
#### (1)、原理
+ 公钥加密，私钥解密
+ 将一个密钥分为一个公钥一个私钥
+ 公钥不可推导出私钥，将公钥公开
+ 数据被接收后用私钥解密得到明文
### (2)、背景
+ 用户群大、分布广，通信效率低
+ 传输密钥的信道无法得到保障
+ 无法证明消息来源正确
## 五、密钥分配与管理
### 5.1、密钥分配的方法
1. 密钥由A选取并通过物理手段发送给B
2. 密钥由第三方选区并通过物理手段发送给A和B
### 5.2、公钥的分配方法
1. 公开发布：用户将自己的公钥发送给每一个用户或进行广播
2. 公用目录表：建立公用的动态目录表，由可信赖的实体或组织承担目录表的建立与维护发布公钥
3. 公钥管理机构：由专门的公钥组织进行管理和分配，受到法律的监督与保护
### 5.3公钥证书
+ **CA：证书管理机构**
### 5.4、密钥托管的定义与功能
#### (1)、原理
1. 将密钥分为t份交给t个托管人
2. 任何一个托管人都无法恢复完整的用户密码
3. 只有这t个人合在一起才能得到完整密钥
#### (2)、功能
1. 防止抵赖
2. 政府监听
3. 用户密码恢复
### 5.5、消息认证
#### (1)、定义
+ 使得接收者也可检验受到消息是否真实
#### (2)、作用
1. 检验消息来源的真实性
2. 检验消息的完整性
3. 检验消息的时效性
###  5.6、HASH函数
+ 提高签名与验证的速度
